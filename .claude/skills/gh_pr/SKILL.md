# GitHub PR Workflow

This skill provides step-by-step workflows for PR operations. Follow the numbered steps exactly.

---

## Creating a PR

### Step 1: Create Feature Branch

```bash
git checkout main && git pull origin main && git checkout -b <username>/<short-description>
```

Branch naming: lowercase with hyphens, prefixed with username (e.g., `bcoloe/add-user-auth`).

### Step 2: Stage and Commit

```bash
git add <specific-files>
git commit -m "$(cat <<'EOF'
<Imperative summary of change>

<Optional detailed explanation>

[Generated by Claude Code]
EOF
)"
```

- Use imperative mood ("Add feature" not "Added feature")
- Keep subject under 72 characters
- Always include `[Generated by Claude Code]` footer

### Step 3: Push and Create PR

```bash
git push -u origin HEAD
gh pr create --title "<Concise title>" --body "$(cat <<'EOF'
## TL;DR
<1-2 sentence summary>

## Changes
- <Bulleted list of changes>

## Testing
- [ ] <Testing checklist>

_[Generated with Claude Code]_
EOF
)"
```

## Addressing PR Comments

### Step 1: Fetch Comments

```bash
.claude/skills/gh_pr/scripts/get-pr-comments.sh
```

This displays all comment threads. Comments without a `[Claude Code]` reply are unaddressed.

### Step 2: Address Each Comment

For each unaddressed comment:
1. Read and understand the feedback
2. Implement the requested change (or prepare explanation if disagreeing)
3. Note the Comment ID for replying later

### Step 3: Commit and Push

```bash
git add <changed-files>
git commit -m "$(cat <<'EOF'
Address PR feedback: <brief summary>

<Details of changes made>

[Generated by Claude Code]
EOF
)"
git push
```

### Step 4: Reply to Comments

For each addressed comment, reply with what was done:

```bash
.claude/skills/gh_pr/scripts/reply-to-comment.sh <COMMENT_ID> "<what you changed>"
```

Replies are auto-prefixed with `[Claude Code]` to track addressed comments.

## Quick Reference

| Action | Command |
|--------|---------|
| View PR | `gh pr view` |
| View PR in browser | `gh pr view --web` |
| List open PRs | `gh pr list` |
| Check CI status | `gh pr checks` |
| View diff | `gh pr diff` |
| Merge (squash) | `gh pr merge --squash` |
| Close PR | `gh pr close` |
