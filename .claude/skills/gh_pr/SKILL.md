The /gh_pr skill provides guidance to support interactions with GitHub Pull Requests (PR). The following subsections describe useful guidance for standard workflows.

## Creating a PR

Follow these steps to create a new PR:

### 1. Create a Feature Branch

Create a branch from the default branch (usually `main`) with a terse, descriptive name:

```bash
git checkout main
git pull origin main
git checkout -b <username>/<short-description>
```

Branch naming conventions:
- Use lowercase with hyphens: `bcoloe/add-user-auth`
- Prefix with username for attribution
- Keep it short but descriptive of intent

### 2. Stage and Commit Changes

Stage changes and create commits with clear messages. Always include the Claude Code attribution in the commit footer:

```bash
git add <specific-files>
git commit -m "$(cat <<'EOF'
<Imperative summary of change>

<Optional detailed explanation of what and why>

[Generated by Claude Code]
EOF
)"
```

Commit message guidelines:
- Use imperative mood in the subject line ("Add feature" not "Added feature")
- Keep subject line under 72 characters
- Separate subject from body with a blank line
- Include `[Generated by Claude Code]` footer on AI-assisted commits

### 3. Push and Create the PR

Push the branch and create the PR with a formatted description:

```bash
git push -u origin HEAD
gh pr create --title "<Concise PR title>" --body "$(cat <<'EOF'
## TL;DR
<Brief 1-2 sentence summary of what this PR accomplishes>

## Changes
- <Bulleted list of specific changes>
- <Include links to relevant code: `src/file.ts:42`>

## Testing
- [ ] <Checklist of testing completed or needed>
- [ ] <Include test commands if applicable>

_[Generated with Claude Code]_
EOF
)"
```

## Reviewing Comments

When asked to address PR comments, follow this workflow:

### 1. Fetch Unresolved Comments

Use the helper script to retrieve all unresolved comment threads:

```bash
.claude/skills/gh_pr/scripts/get-pr-comments.sh [PR_NUMBER]
```

If no PR number is provided, it will use the current branch's PR.

### 2. Address Each Comment

For each comment thread:
1. Read and understand the feedback
2. If you agree with the suggestion, implement the requested change
3. If you fundamentally disagree, prepare a respectful explanation why

### 3. Commit and Push Changes

After addressing comments, commit with a message referencing the feedback:

```bash
git add <changed-files>
git commit -m "$(cat <<'EOF'
Address PR feedback: <brief summary>

<Details of changes made>

[Generated by Claude Code]
EOF
)"
git push
```

### 4. Reply to Comment Threads

Use the helper script to reply to each addressed comment:

```bash
.claude/skills/gh_pr/scripts/reply-to-comment.sh <COMMENT_ID> "<response>"
```

All replies are automatically prefixed with `[Claude Code]` to distinguish AI responses.

Guidelines for replies:
- Be concise and specific about what changed
- Reference the file and line if applicable
- If declining a suggestion, explain the reasoning respectfully

### 5. Identify Unaddressed Comments

Comments without a `[Claude Code]` prefix in their replies have not been addressed. The `get-pr-comments.sh` script filters for these automatically.

## Quick Reference Commands

| Action | Command |
|--------|---------|
| View PR status | `gh pr view` |
| List open PRs | `gh pr list` |
| Check PR checks | `gh pr checks` |
| View PR diff | `gh pr diff` |
| Merge PR | `gh pr merge --squash` |
| Close PR | `gh pr close` |
